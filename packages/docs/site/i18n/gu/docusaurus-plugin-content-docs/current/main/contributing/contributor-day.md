# વર્ડકેમ્પ યોગદાનકર્તા દિવસ

<!-- 
# WordCamp Contributor Day 
-->

વર્ડકેમ્પ યોગદાનકર્તા દિવસ એ એક એવી ઘટના છે જ્યાં વર્ડપ્રેસ સમુદાય વર્ડપ્રેસ પ્રોજેક્ટમાં યોગદાન આપવા માટે ભેગા થાય છે. આ માર્ગદર્શિકા તમે વર્ડપ્રેસ પ્લેગ્રાઉન્ડ પ્રોજેક્ટમાં કેવી રીતે યોગદાન આપી શકો છો અથવા વર્ડપ્રેસ કોરમાં યોગદાન આપવામાં પ્લેગ્રાઉન્ડ તમને કેવી રીતે મદદ કરી શકે છે તેના પર ધ્યાન કેન્દ્રિત કરે છે.

-->

## કોણ યોગદાન આપી શકે છે?

<!--
## Who Can Contribute?
-->

કેટલીક ઇવેન્ટ્સમાં પ્રોજેક્ટ માટે સમર્પિત ટેબલ હશે. વર્ડપ્રેસ પ્લેગ્રાઉન્ડ યોગદાનકર્તા કોષ્ટકો ફક્ત વિકાસકર્તાઓ તરફથી જ નહીં, પરંતુ તમામ પ્રકારના યોગદાનનું સ્વાગત કરે છે. ભલે તમે લેખક, કોડર, ટેસ્ટર, પ્લગઇન અથવા થીમ ડેવલપર, માર્કેટર, સાઇટ માલિક અથવા અન્ય કોઈપણ પ્રકારના વપરાશકર્તા હોવ, તમને યોગદાન આપવા માટે પ્રોત્સાહિત કરવામાં આવે છે.

<!--
Some events will have dedicated tables for projects. WordPress Playground contributor tables welcome not just contributions from developers, but all types of contributions. Whether you are a writer, coder, tester, plugin or theme developer, marketer, site owner, or any other type of user, you are encouraged to contribute.
-->

અમે સમુદાય નિર્માણ, પરીક્ષણ, દસ્તાવેજીકરણ અને ડિઝાઇન સહિત વિવિધ ક્ષેત્રોમાં વિવિધ યોગદાનને મહત્વ આપીએ છીએ।

<!--
We value diverse contributions across different areas including community building, testing, documentation, and design.
-->

# પ્લેગ્રાઉન્ડ પ્રોજેક્ટમાં કેવી રીતે યોગદાન આપવું

<!--
# How to Contribute to the Playground Project
-->

આ વિભાગ વર્ડપ્રેસ પ્લેગ્રાઉન્ડ પ્રોજેક્ટ અને તેના સંકળાયેલ સાધનોમાં સીધા કેવી રીતે યોગદાન આપી શકો છો તેની રૂપરેખા આપે છે:

<!--
This section outlines how you can directly contribute to the WordPress Playground project and its associated tools:
-->

- **દસ્તાવેજીકરણ:** હાલની સામગ્રીમાં સુધારો કરીને, નવી માર્ગદર્શિકાઓ વિકસાવીને અથવા વિવિધ ભાષાઓમાં સામગ્રીનો અનુવાદ કરીને અમારા દસ્તાવેજીકરણને વિસ્તૃત કરો.

<!--
- **Documentation:** Expand our documentation by improving existing content, developing new guides, or translating content into different languages.
-->

- **બ્લુપ્રિન્ટ્સ:** વર્ડપ્રેસ પ્લગઇન રિપોઝીટરીમાં પ્લગઇન્સ માટે પ્લગઇન ડેમો બનાવો, અથવા અમારા પ્રોજેક્ટ દસ્તાવેજીકરણને સમૃદ્ધ બનાવવા માટે નવા બ્લુપ્રિન્ટ્સ વિકસાવો.

<!--
- **Blueprints:** Create plugin demos for plugins in the WordPress plugin repository, or develop new blueprints to enrich our project documentation.
-->

- **પ્લેગ્રાઉન્ડ પર્યાવરણનું પરીક્ષણ:** વર્ડપ્રેસ પ્લેગ્રાઉન્ડ પ્રોજેક્ટનું જ પરીક્ષણ કરવામાં જોડાઓ. તમે જે સમસ્યાઓનો સામનો કરો છો તેનું વર્ણન કરતી નવી સમસ્યાઓ કાળજીપૂર્વક બનાવીને અને કાર્યક્ષમ ઉકેલો સૂચવીને આ કરી શકો છો. અમારા વર્ડપ્રેસ વેબ ઇન્સ્ટન્સ (playground.wordpress.net સાઇટ) નું પરીક્ષણ કરો, અથવા પ્લેગ્રાઉન્ડ દ્વારા સંચાલિત વિવિધ એપ્લિકેશનોનું અન્વેષણ કરો. આ સાધનોનું પરીક્ષણ કરો, તેમની કાર્યક્ષમતાનું અવલોકન કરો અને વિગતવાર પ્રતિસાદ આપો.

<!--
- **Testing the Playground Environment:** Engage in testing the WordPress Playground project itself. You can do this by carefully creating new issues describing the problems you encounter and suggesting efficient solutions. Test our WordPress web instance (playground.wordpress.net site), or explore various applications powered by Playground. Test these tools, observe their functionality, and provide detailed feedback.
-->

- **ઉત્પાદન પ્રતિસાદ:** પ્લેગ્રાઉન્ડ અનુભવને સુધારવા માટે તમારી આંતરદૃષ્ટિ અમૂલ્ય છે. આમાં વેબ ઇન્સ્ટન્સ, એપ્લિકેશન અને કોઈપણ સર્વર-સાઇડ ટૂલ્સ પર સામાન્ય પ્રતિસાદ શામેલ છે.

<!--
- **Product Feedback:** Your insights are invaluable for improving the Playground experience. This includes general feedback on the web instance, applications, and any server-side tools.
-->

રિપોર્ટ કરેલી સમસ્યાઓ અને પરીક્ષણ પરિણામો સહિત તમામ પ્રતિસાદ, અમારા GitHub રિપોઝીટરી દ્વારા સબમિટ કરી શકાય છે.

<!--
All feedback, including reported issues and test results, can be submitted through our GitHub repository.
-->

### ફોલો-અપ અને સતત જોડાણ

<!--
### Follow-up and Continuous Engagement
-->

જ્યારે ઇવેન્ટ દરમિયાન ઘણા કાર્યો પૂર્ણ થાય છે, ત્યારે તમારી યોગદાન યાત્રા ત્યાં સમાપ્ત થવાની જરૂર નથી. કન્ટ્રિબ્યુટર ડે પછી તમારી સમસ્યાઓ પર કામ કરવાનું ચાલુ રાખવા અથવા વિનંતીઓ ખેંચવા માટે તમારું સ્વાગત છે. અમે ઇવેન્ટથી આગળના કાર્યો હાથ ધરતા યોગદાનકર્તાઓ તરફથી ચાલુ પ્રવૃત્તિની અપેક્ષા રાખીએ છીએ. કૃપા કરીને નોંધ લો કે જો પુલ વિનંતી એક મહિના સુધી કોઈ પ્રવૃત્તિ ન બતાવે, તો તેને ત્યજી દેવાયેલી અને ત્યારબાદ બંધ ગણી શકાય.

### મદદ મેળવવી અને જોડાયેલા રહેવું

<!--
### Getting Help and Staying Connected
-->

યોગદાનકર્તા દિવસ દરમિયાન, તમે સમર્પિત પ્લેગ્રાઉન્ડ ટેબલ પર સીધી સહાય મેળવી શકો છો અને અમારી સાથે વાર્તાલાપ કરી શકો છો. સતત સમર્થન અને સમુદાય ક્રિયાપ્રતિક્રિયા માટે, તમે WordPress Slack પર `#playground` ચેનલ પર અથવા GitHub દ્વારા અમારી સાથે જોડાઈ શકો છો.

<!--

## યોગદાનકર્તા દિવસ પર પ્લેગ્રાઉન્ડનો ઉપયોગ કેવી રીતે કરવો

<!--
## How to Use Playground on Contributor Day
-->

હવે અમે કવર કરીશું કે યોગદાનકર્તા દિવસ દરમિયાન પ્લેગ્રાઉન્ડ તમને કેવી રીતે મદદ કરી શકે છે. [વર્ડપ્રેસ પ્લેગ્રાઉન્ડ VS કોડ એક્સટેન્શન](https://marketplace.visualstudio.com/items?itemName=WordPressPlayground.wordpress-playground) અને [@wp-playground/cli](https://www.npmjs.com/package/@wp-playground/cli) સ્થાનિક WordPress વાતાવરણ સેટ કરવાની પ્રક્રિયાને સુવ્યવસ્થિત કરે છે. WordPress પ્લેગ્રાઉન્ડ બંનેને શક્તિ આપે છે - કોઈ ડોકર, MySQL અથવા Apache જરૂરી નથી.

<!--
Now we'll cover how Playground can help you during Contributor Day. The [WordPress Playground VS Code Extension](https://marketplace.visualstudio.com/items?itemName=WordPressPlayground.wordpress-playground) and [@wp-playground/cli](https://www.npmjs.com/package/@wp-playground/cli) streamline the process of setting up a local WordPress environment. Both are powered by WordPress Playground - no Docker, MySQL, or Apache required.
-->

વર્ડપ્રેસમાં યોગદાન આપતી વખતે [સ્થાનિક વિકાસ](/developers/local-development/wp-playground-cli) માટે આ સાધનોનો ઉપયોગ કેવી રીતે કરવો તે શીખવા માટે વાંચતા રહો. કૃપા કરીને નોંધ લો કે એક્સટેન્શન અને NPM પેકેજ વિકાસ હેઠળ છે, અને બધા [WordPress ટીમો બનાવો](https://make.wordpress.org/) સંપૂર્ણપણે સમર્થિત નથી.

<!--
Continue reading to learn how to use these tools for [local development](/developers/local-development/wp-playground-cli) when contributing to WordPress. Please note that the extension and NPM package are under development, and not all [WordPress Make Teams](https://make.wordpress.org/) are fully supported.
-->

## શરૂઆત કરવી

### VS કોડ પ્લેગ્રાઉન્ડ એક્સટેન્શન

<!--
### VS Code Playground Extension
-->

[વિઝ્યુઅલ સ્ટુડિયો કોડ પ્લેગ્રાઉન્ડ એક્સટેન્શન](https://marketplace.visualstudio.com/items?itemName=WordPressPlayground.wordpress-playground) એક મૈત્રીપૂર્ણ શૂન્ય-સેટઅપ વિકાસ વાતાવરણ છે.

<!--
The [Visual Studio Code Playground Extension](https://marketplace.visualstudio.com/items?itemName=WordPressPlayground.wordpress-playground) is a friendly zero-setup development environment.
-->

1. VS કોડ ખોલો અને **એક્સટેન્શન** ટેબ (**જુઓ > એક્સટેન્શન**) પર નેવિગેટ કરો.

<!--
1. Open VS Code and navigate to the **Extensions** tab (**View > Extensions**).
-->

2. સર્ચ બારમાં, _WordPress પ્લેગ્રાઉન્ડ_ લખો અને **ઇન્સ્ટોલ** પર ક્લિક કરો.

<!--
2. In the search bar, type _WordPress Playground_ and click **Install**.
-->

3. પ્લેગ્રાઉન્ડ સાથે ક્રિયાપ્રતિક્રિયા કરવા માટે, **એક્ટિવિટી બાર** માં નવા આઇકોન પર ક્લિક કરો અને **સ્ટાર્ટ વર્ડપ્રેસ સર્વર** બટન દબાવો.

<!--
3. To interact with Playground, click the new icon in the **Activity Bar** and press the **Start WordPress Server** button.
-->

4. થોડીક સેકન્ડોમાં તમારા બ્રાઉઝરમાં એક નવું ટેબ ખુલશે.

<!--
4. Within a few seconds, a new tab will open in your browser.
-->

### @wp-playground/cli NPM પેકેજ

<!--
### @wp-playground/cli NPM Package
-->

[`@wp-playground/cli`](/developers/local-development/wp-playground-cli) એક CLI ટૂલ છે જે તમને એક જ આદેશથી WordPress સાઇટને સ્પિન કરવાની મંજૂરી આપે છે. કોઈ Docker, MySQL, અથવા Apache ની જરૂર નથી.

<!--
[`@wp-playground/cli`](/developers/local-development/wp-playground-cli) is a CLI tool that allows you to spin up a WordPress site with a single command. No Docker, MySQL, or Apache required.
-->

#### પૂર્વજરૂરીયાતો

<!--
#### Prerequisites
-->

`@wp-playground/cli` ને Node.js 20.18 અથવા નવા અને NPM ની જરૂર છે. જો તમે હજુ સુધી નથી કર્યું, તો શરૂ કરતા પહેલા બંને [ડાઉનલોડ અને ઇન્સ્ટોલ કરો](https://nodejs.org/en/download) કરો.

<!--
`@wp-playground/cli` requires Node.js 20.18 or newer and NPM. If you haven't already, [download and install both](https://nodejs.org/en/download) before getting started.
-->

તમે જે Make WordPress ટીમમાં યોગદાન આપો છો તેના આધારે, તમારે તમે ઇન્સ્ટોલ કરેલ કરતાં અલગ Node.js વર્ઝનની જરૂર પડી શકે છે. તમે વર્ઝન વચ્ચે સ્વિચ કરવા માટે Node Version Manager (NVM) નો ઉપયોગ કરી શકો છો. [ઇન્સ્ટોલેશન માર્ગદર્શિકા અહીં શોધો](https://github.com/nvm-sh/nvm#installing-and-updating).

<!--
Depending on which Make WordPress team you're contributing to, you may need a different Node.js version than what you have installed. You can use Node Version Manager (NVM) to switch between versions. [Find the installation guide here](https://github.com/nvm-sh/nvm#installing-and-updating).
-->

#### `@wp-playground/cli` ચલાવી રહ્યા છીએ

<!--
#### Running `@wp-playground/cli`
-->

તેનો ઉપયોગ કરવા માટે તમારે તમારા ઉપકરણ પર `@wp-playground/cli` ઇન્સ્ટોલ કરવાની જરૂર નથી. તમારા પ્લગઇન અથવા થીમ ડિરેક્ટરી પર જાઓ અને નીચેના આદેશો સાથે `@wp-playground/cli` શરૂ કરો:

<!--
You don't need to install `@wp-playground/cli` on your machine to use it. Navigate to your plugin or theme directory and start `@wp-playground/cli` with the following commands:
-->

```bash
cd my-plugin-or-theme-directory
npx @wp-playground/cli@latest server --auto-mount
```

## યોગદાન આપનારાઓ માટે વિચારો

<!--
## Tips for Contributors
-->

### ગુટેનબર્ગ પુલ રિક્વેસ્ટ (PR) બનાવો

<!--
### Create a Gutenberg Pull Request (PR)
-->

1. તમારા GitHub એકાઉન્ટમાં [ગુટેનબર્ગ રીપોઝીટરી](https://github.com/WordPress/gutenberg) ફોર્ક કરો.

<!--
1. Fork the [Gutenberg repository](https://github.com/WordPress/gutenberg) in your GitHub account.
-->

2. પછી, ફાઇલો ડાઉનલોડ કરવા માટે ફોર્ક્ડ રીપોઝીટરી ક્લોન કરો.

<!--
2. Then, clone the forked repository to download the files.
-->

3. જરૂરી ડિપેન્ડન્સીઝ ઇન્સ્ટોલ કરો અને ડેવલપમેન્ટ મોડમાં કોડ બનાવો.

<!--
3. Install required dependencies and build the code in development mode.
-->

```bash
git clone git@github.com:WordPress/gutenberg.git
cd gutenberg
npm install
npm run dev
```

:::info

જો તમે ઉપર સૂચિબદ્ધ પગલાં વિશે અચોક્કસ હોવ, તો સત્તાવાર [ગુટેનબર્ગ પ્રોજેક્ટ કન્ટ્રીબ્યુટર ગાઈડ](https://developer.wordpress.org/block-editor/contributors/) ની મુલાકાત લો. નોંધ કરો કે આ કિસ્સામાં, `@wp-playground/cli` `wp-env` ને બદલે છે.

<!--
If you're uncertain about the steps listed above, visit the official [Gutenberg Project Contributor Guide](https://developer.wordpress.org/block-editor/contributors/). Note that in this case, `@wp-playground/cli` replaces `wp-env`.
-->

:::

એક નવું ટર્મિનલ ટેબ ખોલો, ગુટેનબર્ગ ડિરેક્ટરી પર નેવિગેટ કરો, અને `@wp-playground/cli` નો ઉપયોગ કરીને WordPress શરૂ કરો:

<!--
Open a new terminal tab, navigate to the Gutenberg directory, and start WordPress using `@wp-playground/cli`:
-->

```bash
cd gutenberg
npx @wp-playground/cli@latest server --auto-mount
```

જ્યારે તમે તૈયાર હોવ, ત્યારે GitHub પર તમારા ફોર્ક્ડ રીપોઝીટરીમાં તમારા ફેરફારોને કમિટ કરો અને પુશ કરો અને ગુટેનબર્ગ રીપોઝીટરીમાં પુલ રિક્વેસ્ટ ખોલો.

<!--
When you're ready, commit and push your changes to your forked repository on GitHub and open a pull request to the Gutenberg repository.
-->

### ગુટેનબર્ગ PR નું પરીક્ષણ કરો

<!--
### Test a Gutenberg PR
-->

1. અન્ય ગુટેનબર્ગ PR નું પરીક્ષણ કરવા માટે, તેની સાથે સંકળાયેલ શાખા તપાસો.

<!--
1. To test another Gutenberg PR, check out the branch associated with it.
-->

2. તમારી સ્થાનિક નકલ અપ ટુ ડેટ છે તેની ખાતરી કરવા માટે નવીનતમ ફેરફારો ખેંચો.

<!--
2. Pull the latest changes to ensure your local copy is up to date.
-->

3. આગળ, જરૂરી નિર્ભરતાઓ ઇન્સ્ટોલ કરો, ખાતરી કરો કે તમારું પરીક્ષણ વાતાવરણ નવીનતમ ફેરફારો સાથે મેળ ખાય છે.

<!--
3. Next, install the required dependencies and ensure your testing environment matches the latest changes.
-->

4. અંતે, વિકાસ મોડમાં કોડ બનાવો.

<!--
4. Finally, build the code in development mode.
-->

```bash
# GitHub માંથી branch-name કોપી કરો #
# Copy branch-name from GitHub #

git checkout branch-name
git pull
npm install
npm run dev

# ગુટેનબર્ગ ડિરેક્ટરીની અંદર એક અલગ ટર્મિનલમાં *
# In a separate terminal within the Gutenberg directory *
npx @wp-playground/cli@latest server --auto-mount
```

#### બ્રાઉઝરમાં પ્લેગ્રાઉન્ડ સાથે ગુટેનબર્ગ PR નું પરીક્ષણ કરો

<!--
#### Test a Gutenberg PR with Playground in the Browser
-->

ગુટેનબર્ગ PR નું પરીક્ષણ કરવા માટે તમારે [સ્થાનિક વિકાસ વાતાવરણ](/developers/local-development/) ની જરૂર નથી—તે સીધા બ્રાઉઝરમાં કરવા માટે પ્લેગ્રાઉન્ડનો ઉપયોગ કરો.

<!--
You don't need a [local development environment](/developers/local-development/) to test a Gutenberg PR—use Playground to do it directly in the browser.
-->

1. તમે જે PR નું પરીક્ષણ કરવા માંગો છો તેનું ID કોપી કરો ([ખુલ્લી પુલ વિનંતીઓની સૂચિ](https://github.com/WordPress/gutenberg/pulls) માંથી એક પસંદ કરો).

<!--
1. Copy the ID of the PR you want to test (pick one from the [list of open pull requests](https://github.com/WordPress/gutenberg/pulls)).
-->

2. પ્લેગ્રાઉન્ડનું [ગુટેનબર્ગ PR Previewer](https://playground.wordpress.net/gutenberg.html) ખોલો અને તમે કોપી કરેલ ID પેસ્ટ કરો.

<!--
2. Open Playground's [Gutenberg PR Previewer](https://playground.wordpress.net/gutenberg.html) and paste the ID you copied.
-->

3. એકવાર તમે **Go** પર ક્લિક કરો, પછી Playground PR માન્ય છે કે નહીં તે ચકાસશે અને સંબંધિત PR સાથે એક નવું ટેબ ખોલશે, જે તમને સૂચિત ફેરફારોની સમીક્ષા કરવાની મંજૂરી આપશે.

<!--
3. Once you click **Go**, Playground will verify if the PR is valid and open a new tab with the associated PR, allowing you to review the suggested changes.
-->

## બ્રાઉઝરમાં પ્લેગ્રાઉન્ડ સાથે વર્ડપ્રેસ પ્લગઇન્સનું ભાષાંતર કરો

<!--
## Translate WordPress Plugins with Playground in the Browser
-->

તમે જે પ્લગઇનનો અનુવાદ કરવા માંગો છો તેને લોડ કરીને અને ઇનલાઇન ટ્રાન્સલેશનનો ઉપયોગ કરીને તમે સપોર્ટેડ વર્ડપ્રેસ પ્લગઇન્સનું ભાષાંતર કરી શકો છો. જો પ્લગઇન ડેવલપર્સે વિકલ્પ ઉમેર્યો હોય, તો તમને અનુવાદ વ્યૂના ઉપરના જમણા ટૂલબાર પર **Translate Live** લિંક મળશે. તમે [this Polyglots blog post](https://make.wordpress.org/polyglots/2023/05/08/translate-live-updates-to-the-translation-playground/) પર આ ઉત્તેજક નવા વિકલ્પ વિશે વધુ વાંચી શકો છો.

<!--
You can translate supported WordPress plugins by loading the plugin you want to translate and using inline translation. If plugin developers have added the option, you'll find a **Translate Live** link in the top right toolbar above the translation view. You can read more about this exciting new option at [this Polyglots blog post](https://make.wordpress.org/polyglots/2023/05/08/translate-live-updates-to-the-translation-playground/).
-->
